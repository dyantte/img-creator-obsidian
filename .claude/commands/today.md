# /today — Morning review

Pull together everything relevant to plan my day. Daily notes are auto-generated by the observer script — they contain git commits, active files, and Claude Code session logs. Use that data.

## Steps

1. Read the last 7 daily notes from `Daily/` — these are auto-populated with observations
2. Read all project context files from `Projects/`
3. Look at incomplete tasks (`- [ ]`) across recent daily notes
4. Analyze the git activity and file changes to understand momentum — what projects are hot, what's stale
5. Check for patterns: am I stuck on something? Making progress? Jumping between projects?

## Output

Give me a brief spoken summary:
- **Where I left off** (based on yesterday's observations)
- **What's hot** (projects with recent commits/activity)
- **What's stale** (projects with no activity in days)
- **Carry-over tasks** (incomplete items from recent daily notes)
- **Suggested focus** for today based on momentum

Don't create or modify any files. Just talk to me.
