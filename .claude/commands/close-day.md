# /close-day — End of day processing

End of day wrap-up. Today's daily note already has auto-logged observations (git commits, active files, Claude Code sessions). Use that data to create a summary.

## Steps

1. Read today's daily note from `Daily/` — it has auto-generated observations
2. Read the project context files to understand what the commits/changes mean
3. Check git logs directly for a complete picture of today's work across all projects
4. Scan the vault for connections between today's work and other notes

## Output

Append to today's daily note:

```
## End of Day Summary
### What got done
- [synthesize the git activity and observations into human-readable accomplishments]

### Vault connections
- [any interesting links between today's work and other vault notes — use [[backlinks]]]

### Tomorrow
- [suggested focus based on momentum and what's incomplete]
```

Be concise. The observations are already logged — this section is the synthesis layer on top. Focus on the "so what" not the "what happened."
