# /connect — Bridge two domains

Take two concepts or domains and find connections between them using the vault's content and link graph.

**Domains:** $ARGUMENTS

## Steps

1. Search the vault for all notes related to each domain
2. Map the "neighborhood" of each domain — what notes link to/from each
3. Look for:
   - Direct links between the two domains
   - Shared connections (notes that link to both)
   - Thematic parallels (similar language, similar patterns)
   - Notes where both topics appear together
   - Structural similarities in how I think about each

## Output

### Domain A Neighborhood
- Key notes and themes in this domain

### Domain B Neighborhood
- Key notes and themes in this domain

### Bridges Found
For each connection:
1. **Bridge name** (give it a clear label)
2. **The connection** (how these two domains relate)
3. **Vault evidence** (which notes support this)
4. **Why it matters** (what I could do with this insight)

Find 3-5 bridges. Prioritize surprising or non-obvious connections over surface-level similarities.
